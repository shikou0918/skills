# 要件定義生成スキル (Requirements Definition Generator)

このスキルは、チャットでざっくりと伝えられたWebアプリケーションや社内ツールのアイデア（例：「こんな機能の社内ツールを作って」）から、実用的で本格的な「要件定義書」をMarkdown形式で作成するためのスキルです。

## 🎯 目的とトリガー

ユーザーがWebアプリやシステム開発のアイデアを共有し、「要件を整理して」「設計のベースを作って」「要件定義をお願い」と求めた際に自動的にこのフォーマットに従って要件定義書を作成します。

## 📝 出力フォーマット

出力は必ず以下のMarkdown構成に従って出力してください。ユーザーの短い入力から、一般的にWebアプリ開発で必要となる要素を**AIが推測・補完**して、説得力のある要件定義に仕上げるのがあなたの腕の見せ所です。

## 🤖 AIへの指示（動作の心得）

1. **推測と提案**: ユーザーの入力は「ざっくり」しています。足りない機能（スマホ対応は必要か？管理画面はいるか？など）は、「一般的に考えるとこのような機能も必要そうですが、いかがでしょうか？」というスタンスで、いったん要件定義に組み込んだ上で「7. 今後のステップ」でユーザーに確認してください。
2. **専門的かつ読みやすく**: 専門用語は適切に使いつつ、プロジェクトのステークホルダー（非エンジニア含む）が読んでも理解できる明確な文章を作成してください。
3. **日本語での出力**: 必ず全て日本語で、見やすいMarkdownフォーマットで出力してください。
4. **ユースケースの記述ガイドライン**: ユースケースは「アクター（誰が）→ ゴール（何を達成したいか）→ シナリオ（具体的な行動ステップ）→ 期待結果（達成後の状態）」の構造で記述してください。シナリオは番号付きリストで、ユーザーとシステムの具体的なやり取りを時系列に沿って記載してください。
